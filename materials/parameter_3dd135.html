3DD135. Единственный корень на отрезке [0; 1]

Нужно найти все значения параметра a, при которых уравнение имеет ровно один корень на отрезке.
(2x + ln(x + 2a))² = (2x − ln(x + 2a))²,   x ∈ [0; 1]
Цель: подготовка к ЕГЭ
Уровень: уверенный
Стиль: как репетитор
Метод: только школьные методы
Автор: Лёвин Артём Александрович

Идея решения

  1. Заметим, что равенство квадратов означает: либо выражения равны, либо они противоположны.
  2. Из этого сразу появляются два кандидата на корни: x = 0 и x = 1 − 2a.
  3. Дальше остаётся проверить только две вещи: область определения логарифма и попадание в отрезок [0;1].
  4. Потом смотрим, когда допустим ровно один корень, а не два.
Это очень типичная идея для задач вида U² = V²: перейти к случаям U = V или U = −V.

Подробное решение

Можно раскрывать шаги по очереди. При наведении на шаг подсвечивается связанный элемент на модели справа.

Ответ

a = 0 или a ≥ 1/2.

Финальная проверка ответа

  • При a = 0 корень x = 0 не подходит, потому что ln(0) не определён. Но корень x = 1 подходит. Значит, корень на отрезке ровно один.
  • При 0 < a < 1/2 подходят оба корня: x = 0 и x = 1 − 2a ∈ (0;1). Значит, корней два — это не подходит.
  • При a = 1/2 оба случая дают один и тот же корень x = 0. Значит, корень единственный.
  • При a > 1/2 корень x = 1 − 2a уходит левее нуля, а x = 0 остаётся допустимым. Значит, снова ровно один корень.
  • При a < 0 корень x = 0 недопустим, а x = 1 − 2a > 1, то есть не лежит на отрезке. Корней нет.

Типичные ошибки

  • Сразу извлечь корень и написать 2x + ln(x+2a) = 2x − ln(x+2a), забыв случай U = −V.
  • Не проверить область определения логарифма: нужно x + 2a > 0.
  • Посчитать, что при a = 1/2 корней два, хотя оба случая дают один и тот же корень x = 0.
  • Проверить только формулы для корней, но забыть условие x ∈ [0;1].
  • Неверно обработать случай a = 0: корень x = 0 запрещён, потому что логарифм не существует.

Самопроверка

Что обязательно нужно проверить в самом конце?
Нужно отдельно проверить: область определения, попадание в отрезок, совпадение/различие корней и отсутствие других случаев.
Почему вообще появляются только два кандидата на корни?
Потому что из равенства квадратов следует только два случая: U = V или U = −V. Больше никаких возможностей нет.
Чем опасен случай a = 0?
Тем, что формально x = 0 возникает из одного из случаев, но логарифм ln(x + 2a) = ln(0) не существует.

Краткая общая идея

Для шаблона (mx + ln(x + ka))² = (mx − ln(x + ka))² всегда получаем два кандидата: x = 0 и x = 1 − ka.
