Решение задания с параметром
(x + ln(x + a))² = (x − ln(x + a))²

Фокус этой страницы — не просто получить ответ, а увидеть структуру уравнения: равенство квадратов превращается в две симметричные ветви, а область допустимых значений отсекает неверные ходы. Справа — интерактивная модель с координатной плоскостью, ветвями, доменом, зумом и динамической проверкой.

Цель: подготовка к ЕГЭ Уровень: уверенный Подача: как репетитор Фокус: логика отбора случаев Режим: пошаговое раскрытие

Идея решения

  • Обозначим A = x + ln(x + a), B = x − ln(x + a). Тогда дано равенство A² = B².
  • Для равенства квадратов удобно не раскрывать всё вслепую, а помнить эквивалентность: A² = B² ⇔ A = B или A = −B.
  • Обе ветви очень просты: из первой получаем ln(x + a) = 0, из второй — x = 0.
  • После этого остаётся аккуратно проверить ОДЗ: логарифм существует только при x + a > 0.
  • Ещё один важный момент: две найденные ветви могут дать один и тот же корень. Это происходит при a = 1.

Пошаговое решение

1 Что означает условие и где вообще можно решать

В уравнении есть выражение ln(x + a), значит сначала обязаны записать область допустимых значений:

x + a > 0
Это не дополнительное украшение, а обязательная часть решения. Любой найденный x нужно проверить на выполнение этого неравенства.

На модели справа граница ОДЗ показана вертикальной линией x = −a. Решать можно только справа от неё.

Если забыть ОДЗ, можно механически подставить x = 0 и получить формально красивый ответ, который на самом деле не существует при a ≤ 0, потому что тогда ln(a) не определён. Поэтому ОДЗ идёт раньше отбора корней.

2 Геометрическая и алгебраическая интерпретация равенства квадратов

Обозначим:

A = x + ln(x + a), B = x − ln(x + a)

Тогда уравнение принимает вид

A² = B²

А это эквивалентно двум случаям:

A = B или A = −B
Именно это — ключевая идея. Мы заменяем одно «тяжёлое» уравнение двумя очень простыми ветвями.

На рисунке справа эти две ветви подписаны. Такой взгляд особенно полезен в задачах с параметром: сразу видно, какие случаи надо разбирать отдельно.

Можно было бы раскрыть квадраты и сократить одинаковые части. Это тоже работает. Но для понимания структуры задачи ветвление A = ±B обычно полезнее: сразу читается симметрия выражения относительно знака перед логарифмом.

3 Первая ветвь: A = B

x + ln(x + a) = x − ln(x + a)

Вычтем x из обеих частей:

ln(x + a) = −ln(x + a)

Перенесём всё влево:

2ln(x + a) = 0

Значит

ln(x + a) = 0

Отсюда

x + a = 1 ⇒ x = 1 − a
Из первой ветви мы всегда получаем кандидат x = 1 − a. Он автоматически допустим, потому что тогда x + a = 1 > 0.

Это очень приятный момент: после перехода от логарифма к аргументу получаем постоянное значение 1. Поэтому здесь не возникает дополнительной зависимости допустимости от параметра.

4 Вторая ветвь: A = −B

x + ln(x + a) = −(x − ln(x + a))

Раскроем скобки справа:

x + ln(x + a) = −x + ln(x + a)

Сократим одинаковые логарифмы:

x = −x

Значит

2x = 0 ⇒ x = 0
Но тут возникает проверка по ОДЗ: при x = 0 логарифм существует только если a > 0.

То есть корень x = 0 появляется не всегда, а только при положительных a.

Это типичный момент для параметров: формальный алгебраический кандидат найден, но его существование зависит от области допустимых значений. Именно поэтому итоговый ответ нельзя писать без разбора по a.

5 Собираем все кандидаты и смотрим, когда они совпадают

Из ветви A = B x = 1 − a, и этот корень допустим всегда.
Из ветви A = −B x = 0, но только если a > 0.

Теперь проверим, не дают ли эти ветви один и тот же корень:

1 − a = 0 ⇒ a = 1

Значит при a = 1 оба случая приводят к одному и тому же решению x = 0.

Итак, совпадение ветвей бывает только при a = 1. Во всех остальных допустимых случаях это разные корни.

Это особенно важно, если в исходной формулировке спрашивают количество корней. Тогда при a = 1 нужно не записывать «два одинаковых корня», а понимать, что решение одно.

6 Ответ по случаям

Если a ≤ 0 Только один корень: x = 1 − a.
Если a = 1 Только один корень: x = 0.
Если a > 0 и a ≠ 1 Два корня: x = 1 − a и x = 0.
Эквивалентная компактная запись: всегда есть корень x = 1 − a; дополнительно корень x = 0 существует только при a > 0; при a = 1 эти два выражения совпадают.

7 Явная проверка ответа

  • Подставим x = 1 − a: тогда x + a = 1, значит ln(x + a) = ln 1 = 0. Обе части становятся (1 − a)². Подходит.
  • Подставим x = 0: получаем (ln a)² = (−ln a)², это верно, но только если a > 0, чтобы ln a существовал.
  • Других корней нет, потому что уравнение эквивалентно двум ветвям A = B и A = −B, а каждая из них уже решена полностью.
Проверка здесь нужна не формально: она отдельно показывает, почему найденные значения подходят и почему больше ничего появиться не может.

8 Краткий вывод

В задачах вида (U)² = (V)² очень выгодно сразу переходить к случаям U = V и U = −V. Здесь это даёт два прозрачных кандидата: x = 1 − a и x = 0. Затем ОДЗ и совпадение ветвей при a = 1 полностью завершают задачу.

Типичные ошибки

  • Забыть ОДЗ и автоматически оставить корень x = 0 при a ≤ 0.
  • После A² = B² писать только A = B и потерять вторую ветвь A = −B.
  • Считать, что при a = 1 есть два корня, хотя обе ветви дают один и тот же x = 0.
  • После ln(x + a) = 0 записать x + a = 0 вместо правильного x + a = 1.
  • Смешать «кандидаты из ветвей» и «окончательный ответ после проверки», не разделяя эти этапы.

Мини-квиз

Самопроверка

Проверьте себя без подсказки: при каком значении параметра два полученных кандидата совпадают?

Они совпадают при 1 − a = 0, то есть при a = 1.

Быстрая проверка логики: почему корень x = 1 − a допустим для любого a?

Потому что при x = 1 − a получаем x + a = 1 > 0, значит логарифм всегда существует.

Контрольный вопрос: почему нельзя ограничиться одной формулой x·ln(x+a)=0 и сразу записать два корня без разбора?

Потому что корень x = 0 требует дополнительной проверки ОДЗ: нужен ln(a), а значит a > 0.

Похожие задачи

Генератор сохраняет тот же метод: равенство квадратов сводится к двум ветвям или к произведению, а затем идёт проверка ОДЗ.

Итоговый ответ

x = 1 − a — всегда; x = 0 — только при a > 0

Если нужно записать по количеству корней, то:

  • при a ≤ 0 — один корень x = 1 − a;
  • при a = 1 — один корень x = 0;
  • при a > 0, a ≠ 1 — два корня: x = 1 − a и x = 0.
Печатная версия сохраняет полный разбор, формулы и итоговые случаи без управляющих элементов.
