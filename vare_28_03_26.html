Обзор темы

В этом блоке — короткий маршрут по всей главе. Он нужен, чтобы быстро вспомнить, какой метод запускается первым и какие ошибки чаще всего ломают решение.

Иррациональные уравнения

√f(x) = g(x) ⇒ сначала g(x) ≥ 0, потом f(x) = g(x)²
Главная опасность: лишние корни после возведения в квадрат.

Показательные уравнения

8x, 4x, 2x+4 ⇒ всё к основанию 2, затем t = 2x
Главная опасность: забытый фильтр t > 0.

Логарифмические уравнения

log5(2 − x) = log25(x⁴) ⇒ общая база + ОДЗ + равенство аргументов
Главная опасность: отбросить логарифмы до проверки области определения.

Визуальная лаборатория

Переключай тип уравнения и включай слои. Это не просто иллюстрации: здесь зашита логика правильного первого шага.

В этом типе уравнений решает не «квадрат», а фильтрующее ограничение, записанное до квадрата.
Ключевая идея:
√f(x) = g(x) ⇔ { g(x) ≥ 0; f(x) = g(x)² }
1
Сначала проверяем правую часть

Корень слева не может быть отрицательным, значит правая часть тоже должна быть неотрицательной.

2
Только потом возводим в квадрат

Иначе можно получить лишние корни, которые формально решают квадратное уравнение, но не исходное.

3
Финальная фильтрация обязательна

В нашем примере число 5 появилось после разложения, но не прошло условие x ≤ 3.

Переключатели слева позволяют буквально увидеть, как решение строится слоями: ограничение → квадрат → проверка.
t = 4.0 P(t) = t³ − 7t² − 16t + 112
Точка движется по оси t. Хорошо видно, что отрицательный корень t = −4 сразу выбывает из-за условия t > 0.
8x − 7·4x − 2x+4 + 112 = 0
↓
(2x)³ − 7(2x)² − 16·2x + 112 = 0
↓
t³ − 7t² − 16t + 112 = 0
1
Всё к одной базе

Если удаётся переписать все слагаемые через 2x, значит замена почти неизбежна и почти всегда полезна.

2
Записываем t > 0

Это экономит ошибки: корни t = 0 или t < 0 не могут дать значения 2x.

3
Возвращаемся к x только в самом конце

Сначала решаем алгебру по t, потом делаем обратную замену: 2x = 4, 2x = 7.

P(4.0) = 0
Эта схема показывает, как «смешанные основания» сводятся к одному отношению t = (3/2)x.
Теорема Безу в этом блоке особенно полезна.
Если P(3) = 0, то P(t) делится на (t − 3).
Для P(t) = t³ − 13t² + 42t − 36 получаем:
P(t) = (t − 3)(t² − 10t + 12)
1
Не боимся двух оснований

Если в уравнении есть 3x и 2x, это не повод отступать. Часто их отношение гораздо важнее самих степеней.

2
Деление нужно обосновать

Мы делим только потому, что 2x всегда положительно. Это обязательная часть корректного решения.

3
Финальный переход

(3/2)x = t ⇒ x = log по основанию 3/2 от t. Затем остаётся сравнить корни с нужным отрезком.

Введите значение x и проверьте, проходит ли оно через ОДЗ.
Полезная цепочка преобразований:
log25(x⁴) = ½log5(x⁴) = log5(x²)
1
У логарифма нет права на небрежность

Аргумент каждого логарифма должен быть строго положительным. Ноль и отрицательные значения запрещены.

2
Общая база делает уравнение прозрачным

Как только база становится одинаковой, можно приравнять аргументы и перейти к обычному квадратному уравнению.

3
Проверка остаётся последним фильтром

Даже если корень получился красиво, он обязан пройти условия 2 − x > 0 и x⁴ > 0.

Теория в карточках

Здесь собраны короткие, но сильные правила. Раскрывайте только то, что нужно прямо сейчас.

√f(x) = g(x) ⇔ { g(x) ≥ 0; f(x) = g(x)² }
Нельзя сразу возводить в квадрат, не проверив знак правой части. Именно это условие отсеивает лишние корни, которые почти всегда появляются в сложных иррациональных уравнениях.
√f(x) = √g(x) ⇔ { f(x) ≥ 0; f(x) = g(x) } или { g(x) ≥ 0; f(x) = g(x) }
Не нужно усложнять себе жизнь. Достаточно взять то ограничение, которое проще вычислять. После этого подкоренные выражения можно приравнять.
8ˣ = (2ˣ)³, 4ˣ = (2ˣ)², 2ˣ⁺⁴ = 16·2ˣ
После этого естественная замена:
t = 2ˣ, t > 0
Важно не потерять условие t > 0. Это не мелочь, а часть решения.
Обозначаем:
a = 3ˣ, b = 2ˣ
Получаем многочлен по a и b. Потом делим на положительное выражение b³ и переходим к отношению:
t = a / b = (3/2)ˣ
Это превращает громоздкое показательное уравнение в обычное алгебраическое.
Если для многочлена P(t) выполнено P(c) = 0, то P(t) делится на t − c.
P(t) = t³ − 13t² + 42t − 36, P(3) = 0 ⇒ P(t) делится на t − 3
Это позволяет расколоть кубический многочлен на линейный и квадратный множители.
Аргумент логарифма должен быть строго положительным. Поэтому уравнение
log₅(2 − x) = log₂₅(x⁴)
начинается не с преобразования базы, а с ограничений:
2 − x > 0 и x⁴ > 0
Только потом можно переходить к равенству аргументов.

Разбор задач

Ниже — два полноценных решения. Они построены не как «ответы», а как образцы хода мысли.

Разбор 1 · иррациональное уравнение

√(x³ − 4x² − 10x + 29) = 3 − x

Шаг 1. Записываем фильтр.
3 − x ≥ 0, значит x ≤ 3.
Шаг 2. Возводим в квадрат только после фильтра.
x³ − 4x² − 10x + 29 = (3 − x)² = 9 − 6x + x²
Шаг 3. Переносим всё влево.
x³ − 5x² − 4x + 20 = 0
Шаг 4. Группируем.
(x³ − 5x²) + (−4x + 20) = 0
x²(x − 5) − 4(x − 5) = 0
(x − 5)(x² − 4) = 0
Шаг 5. Находим кандидатов.
x = 5, x = 2, x = −2
Шаг 6. Возвращаемся к фильтру x ≤ 3.
x = 5 не подходит, x = 2 и x = −2 подходят.
Показать итог и комментарий
Ответ: x = −2, x = 2
Если дополнительно нужен отрезок [−√3; √30], то в него попадает только x = 2, потому что −2 < −√3.
Разбор 2 · показательное уравнение

8ˣ − 7·4ˣ − 2ˣ⁺⁴ + 112 = 0

Шаг 1. Всё приводим к основанию 2.
8ˣ = (2ˣ)³, 4ˣ = (2ˣ)², 2ˣ⁺⁴ = 16·2ˣ
Шаг 2. Делаем замену.
t = 2ˣ, t > 0
Шаг 3. Получаем кубическое уравнение.
t³ − 7t² − 16t + 112 = 0
Шаг 4. Разложение группировкой.
(t³ − 7t²) + (−16t + 112) = 0
t²(t − 7) − 16(t − 7) = 0
(t − 7)(t² − 16) = 0
Шаг 5. Решаем по t.
t = 7, t = 4, t = −4
Шаг 6. Учитываем t > 0 и возвращаемся к x.
t = −4 не подходит.
2ˣ = 4 ⇒ x = 2
2ˣ = 7 ⇒ x = log по основанию 2 от 7
Показать итог и комментарий
Ответ: x = 2 и x = log27
На отрезке [log25; log211] лежит только log27. Число 2 — это log24, а 4 < 5.

Cheat sheet

Компактная сводка правил, которую удобно просмотреть перед самостоятельной работой.

Корни и равносильность

  • √f(x) = g(x) ⇒ сначала g(x) ≥ 0, потом квадрат.
  • √f(x) = √g(x) ⇒ можно взять одно удобное ограничение и приравнять подкоренные выражения.
  • После квадрата обязательно проверяй кандидатов.

Показательные замены

  • Ищи общий фрагмент: 8ˣ, 4ˣ, 2ˣ⁺⁴ почти кричат о замене t = 2ˣ.
  • Всегда дописывай t > 0.
  • Обратная замена — только после полного решения по t.

Смешанные основания

  • Если есть 3ˣ и 2ˣ, попробуй a = 3ˣ, b = 2ˣ.
  • Деление на b³ допустимо, если заранее показано, что b > 0.
  • Часто всё сводится к t = (3/2)ˣ.

Логарифмы

  • Сначала ОДЗ: аргументы логарифмов строго больше нуля.
  • Потом приводи к одной базе.
  • Только после этого переходи к равенству аргументов.
